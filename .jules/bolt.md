## 2025-04-06 - Fixing Database Network Waterfalls
**Learning:** In Supabase applications, sequentially executing `await supabase.from(...)` statements creates severe network waterfalls where each query blocks the next, leading to significantly inflated load times and slow state mutations. This is particularly problematic in functions like `loadUserData` and `updateSkillProgress` where multiple distinct tables (`user_learning_profiles`, `user_skills`, `user_topics`) need to be read or written simultaneously.
**Action:** Always batch independent Supabase queries concurrently using `Promise.all()`. For mutations, avoid redundant database reads by calculating intermediate state (like completion counts or progress percentages) locally based on the existing React context state before dispatching the concurrent write batch.
