## 2024-06-19 - DOM-based XSS via CSS Injection in Shadcn UI Chart Component
**Vulnerability:** The Shadcn UI `ChartContainer` component dynamically generated CSS inside a `<style dangerouslySetInnerHTML={{...}} />` tag. The `id` prop was directly interpolated into the CSS string without sanitization. An attacker could potentially inject malicious CSS or escape the style tag if the `id` was controlled by user input.
**Learning:** Components that rely on string interpolation for dynamically generated styles inside `dangerouslySetInnerHTML` are prone to CSS injection and potentially DOM-based XSS if attributes can be closed prematurely.
**Prevention:** Always strictly sanitize dynamic properties (like `id` attributes) before interpolating them into HTML strings or CSS selectors. In this case, `id.replace(/[^a-zA-Z0-9-]/g, "")` was used to ensure only safe characters are included in the generated ID.
## 2024-04-10 - Secure Logging in Supabase Edge Functions
**Vulnerability:** Raw external API responses, generated AI content, and complete error objects were being logged directly to `console.error` in the `generate-content` edge function. This created a severe risk of information leakage, exposing API keys (if contained in error messages), unexpected internal response structures, or potentially sensitive unparsed outputs to monitoring tools.
**Learning:** In Deno-based Supabase Edge Functions, `console.error` logs are aggregated and potentially visible in dashboards or logs exports. Exposing full `e` objects instead of using `e instanceof Error ? e.message : 'Unknown'` bypasses standard security boundaries by letting external tools dump unchecked memory references and full stack traces.
**Prevention:** Always validate and sanitize inputs before logging them. Prefer specific fields (like `response.status`) over full `response.text()`, avoid logging generated or raw text payloads directly unless wrapped in length-limiting sanitizers, and use explicit `instanceof Error` checks when dealing with caught exceptions in TypeScript/Deno to avoid implicit `any` behavior exposing unexpected properties.
