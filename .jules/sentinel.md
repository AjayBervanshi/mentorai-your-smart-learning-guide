## 2025-04-05 - Fix potential XSS via CSS Injection in Chart component
**Vulnerability:** The reusable `ChartContainer` component from shadcn/ui accepted an `id` prop which was directly interpolated into a `chartId` string. This `chartId` was then passed into `dangerouslySetInnerHTML` inside `ChartStyle`. If a developer passes unsanitized user input into the `id` prop, an attacker could break out of the CSS selector (e.g., using `] { ... } </style><script>alert(1)</script>`) and execute arbitrary JavaScript or inject malicious CSS.
**Learning:** Reusable UI components that inject identifiers into `<style>` tags or HTML attributes using `dangerouslySetInnerHTML` are a common vector for DOM-based XSS if the identifier can be controlled by user input.
**Prevention:** Always sanitize dynamic identifiers injected into HTML strings or `<style>` blocks. In this case, removing any non-alphanumeric and non-hyphen characters (`id.replace(/[^a-zA-Z0-9-]/g, "")`) ensures the ID remains a valid and safe CSS selector/DOM ID.
